## #15 프로젝트간 자금이동

**개념:** 한 프로젝트의 자금을 다른 프로젝트로 이동할 때 전표 2장을 묶어서 발행.

```
A 프로젝트 → 개인-인출 (보통예금 대변 / 개인 차변)
B 프로젝트 → 개인-투입 (개인 대변 / 보통예금 차변)
```

- 두 전표를 하나의 "자금이동" 트랜잭션으로 묶어 관리
- UI: 출처 프로젝트 / 목적지 프로젝트 / 금액 / 날짜 입력 → 전표 2장 자동 발행
- 취소 시 두 전표 동시 취소

## #16 마이너스통장 (한도대출) 관리

**운용 구조:**
- 마이너스통장 전체 한도 안에서 각 프로젝트에 배분
  - 예) 한도 5천만: A프로젝트 3천 / B프로젝트 5백 / 일반 나머지
- 프로젝트 배분분(A, B)은 `장기차입금`으로 처리하여 **고정 이자** 발생
- 일반분은 수시 인출/상환으로 **유동 이자** 발생 (잔액이 매일 변동)

**전표 흐름:**
1. 마통 인출 → 장기차입금↑(해당 프로젝트) / 보통예금↑(일반)
2. 자금이동(#15) → 일반에서 해당 프로젝트로 이동
3. 매월 이자 → A·B는 고정금액 (장기차입금 잔액 기준), 일반은 일할계산

**DB 설계 방향 (미확정):**
- `loans` 테이블에 `loan_type='마이너스통장'` 추가
- 프로젝트별 배분 잔액 별도 테이블 or loans row 분리
- 일반분은 기존 일할이자(daily_actual) 방식 그대로 적용
- A·B분은 현재 만기일시 방식과 유사 (잔액 고정, 월정액 이자)
