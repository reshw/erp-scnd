| **대출표** | 대출 상환 스케줄 |
| **CFM** | 현금흐름 예측 |

### 복식부기 핵심 로직

```
현금출납 컬럼: 전표번호 | 날짜 | 분류 | 법인 | 프로젝트 | 계정과목 | 차변 | 대변 | 거래처 | 적요
```

- 하나의 `전표번호`에 **2개 이상의 행** (차변 합 = 대변 합)
- `분류`는 `분류표`의 "활동구분 - 유형" 형식 (예: `현금 - 입금`, `투자 - 취득`, `개인 - 투입`)
- **활동구분**: 영업, 재무, 투자, 개인, 현금, 세무

## DB 스키마 설계

### 테이블 구조

```
accounts        ← 계정과목 마스터
projects        ← 프로젝트 (JH308, JH407 등)
entities        ← 법인/사업체
counterparties  ← 거래처 DB
bank_accounts   ← 통장/계좌
journals        ← 전표 헤더 (전표번호)
journal_lines   ← 전표 명세 (복식부기 핵심)
loans           ← 대출 정보
```

### Views

- `account_balances` - 계정별 잔액
- `project_balances` - 프로젝트별 잔액
- `monthly_cashflow` - 월말마감 집계
