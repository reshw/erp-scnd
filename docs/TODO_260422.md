# TODO 260422 - 마이너스통장(한도대출) #16

## 개념 정리

마이너스통장 전체 한도 안에서 두 가지 방식으로 운용:

| 구분 | 특성 | 이자 계산 |
|------|------|-----------|
| 프로젝트 배분분 (A, B…) | 한도의 일부를 특정 프로젝트에 배정, 잔액 고정 | 잔액 × 연이율 / 12 (월정액, 만기일시 방식) |
| 일반분 | 수시 인출/상환, 잔액 매일 변동 | 일별잔액 × 연이율 / 365 × 실일수 합산 |

전표 흐름:
```
인출 → 장기차입금↑ / 보통예금↑
자금이동(#15) → 일반 보통예금에서 프로젝트 보통예금으로
이자납부 → 이자비용↑ / 보통예금↓
```

## 설계 확정

### 잔액 추적: 전표 기반 역산 (별도 draws 테이블 없음)

`journal_lines`에서 조건 필터:
- `account_id` = 장기차입금 계정
- `counterparty_id` = 해당 마통 거래처 (예: 대출-마통경남)

```
대변(credit) 합산 → 누적 인출액
차변(debit)  합산 → 누적 상환액
일별잔액     = 날짜순 누적 (credit - debit)
월이자       = Σ (일별잔액 × 연이율 / 365) for each day in month
```

전표가 단일 진실. 별도 테이블 불필요.

### DB 구조: loans row 분리 (안 B)

| row | 용도 | loan_type | interest_calc |
|-----|------|-----------|---------------|
| 마통-A프로젝트 | 고정 배분분 | 만기일시 | monthly |
| 마통-B프로젝트 | 고정 배분분 | 만기일시 | monthly |
| 마통-일반 | 유동분 | 마이너스통장 | daily_actual |

- A·B 배분분: 기존 loan 스케줄 + settlements 그대로 사용
- 일반분: 신규 `마이너스통장` loan_type + 전표 역산 방식

### 미결 사항 — 전부 확정

**Q1. 한도 초과 경고** — 불필요. 한도 없으면 애초에 인출 전표 자체가 안 나감.

**Q2. 당일 이자 발생 기준** — `include_draw_day` 컬럼(boolean)으로 ON/OFF 옵트아웃. 기본 ON(당일 포함).

## DB 설계

```sql

ALTER TABLE loans
  ADD COLUMN IF NOT EXISTS overdraft_limit numeric;
-- loan_type CHECK에 '마이너스통장' 추가 (현재 CHECK 없으면 그냥 text로 저장)
```

이자 계산 쿼리 (서버사이드):
```sql
-- 기간 내 journal_lines에서 일별 잔액 구성
SELECT date, SUM(credit - debit) OVER (ORDER BY date) AS balance
FROM journal_lines
WHERE account_id = $account_id
  AND counterparty_id = $counterparty_id
  AND date BETWEEN $start AND $end
ORDER BY date
```
